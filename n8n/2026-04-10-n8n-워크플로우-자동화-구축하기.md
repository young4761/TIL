### Context
n8n을 사용하여 디스코드에서 메시지를 수신한 후 자동으로 TIL 형식으로 변환하고 GitHub에 푸시하는 워크플로우를 구축했습니다. 이 워크플로우는 메시지를 수신하면 에이전트가 이를 처리하고, 최종적으로 README.md 파일을 업데이트하며 완료 메시지를 발신합니다.

### Core
```json
define_workflow {
  "nodes": [
    {
      "id": "1",
      "type": "Discord Trigger",
      "parameters": {
        "webhookUrl": "YOUR_DISCORD_WEBHOOK_URL"
      }
    },
    {
      "id": "2",
      "type": "OpenAI",
      "parameters": {
        "model": "gpt-3.5-turbo",
        "prompt": "Transform the following text into TIL format: {{input_text}}",
        "temperature": 0.7
      }
    },
    {
      "id": "3",
      "type": "GitHub",
      "parameters": {
        "action": "create_or_update_file",
        "repository": "your_repo",
        "file": "your_file_name.md",
        "content": "{{output_text}}"
      }
    },
    {
      "id": "4",
      "type": "HTTP Request",
      "parameters": {
        "url": "YOUR_REPO_URL/README.md",
        "method": "POST",
        "body": "Your commit message"
      }
    },
    {
      "id": "5",
      "type": "Discord Message Sender",
      "parameters": {
        "message": "Workflow completed successfully!"
      }
    }
  ],
  "connections": {
    "1": [
      { "node": "2" },
      { "node": "4" }
    ],
    "2": [
      { "node": "3" }
    ],
    "3": [
      { "node": "5" }
    ]
  }
}
```

### Insight
n8n을 처음 사용해본 결과, 기본적인 입력과 출력을 이해하는 것이 워크플로우 구축의 핵심임을 깨달았습니다. 다양한 작업을 손쉽게 자동화할 수 있는 방식과 간단한 노드 설정으로 복잡한 프로세스를 관리할 수 있었습니다. n8n의 친절한 커뮤니티가 큰 도움이 되었습니다. 특히, 후츠릿의 도움을 감사드립니다.
